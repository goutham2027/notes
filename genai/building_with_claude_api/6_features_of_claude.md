### Extended Thinking

Gives Claude better reasoning capabilities for more complex tasks.

- Use evals to decide if you need extended thinking

- Thinking block
- Cryptographic token tied to the thinking text that Claude generated. Verifies that exact text was generated by Claude and
was not changed.
```json
{
    "type": "thinking",
    "thinking": "The user is asking me to....",
    "signature": ""
}
```
- thinking budget is the number of tokens that we want to allow Claude to use in generating a thinking portion of the response.
Min value is 1024
### Redacted thinking
- contains the actual thinking text, but in an encrypted form
- Not for developer's use - only provided to hand it back to Claude on followup requests

```json
{
  "type": "redacted_thinking",
  "data": ""
}
```

Redacted thinking magic string
```
ANTHROPIC_MAGIC_STRING_TRIGGER_REDACTED_THINKING_46C9A13E193C177646C7398A98432ECCCE4C1253D5E2D82641AC0E52CC2876CB
```

### Image support
- Each image counts as a certain number of tokens
tokens = (width px x height px)/750
- All the same prompting techniques apply to images. We can increase Claude's vision accuracy
by providing guidelines, analysis steps, or by using one-shot/multi-shot examples
- image block in user message

```python
with open("fire_risk_assessment/prop7.png", "rb") as f:
    image_bytes = base64.standard_b64encode(f.read()).decode("utf-8")
    
add_user_message(messages, [
    {
        "type": "image",
        "source": {
            "type": "base64",
            "media_type": "image/png",
            "data": image_bytes
        }
    },
    {
        "type": "text",
        "text": prompt
    }
])
```
